# Self-Review Checklist

Before I open a Pull Request, I have verified the following:

- [ ] **Code Correctness**: Does the code do exactly what the title and description claim?
- [ ] **Test Coverage**: Have I run the tests locally and do they all pass?
- [ ] **Documentation**: Is the README accurate and updated with any new steps?
- [ ] **Scope**: Is this PR focused on one specific task or feature (no "scope creep")?
- [ ] **No Debug Artifacts**: Have I removed all `print()`, `breakpoint()`, or temporary comments?
